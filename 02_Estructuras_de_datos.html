sp: Classes and methods for spatial data

Clases en sp

La librería da soporte a las clases:

  • CRS
  • SpatialPoints*
  • SpatialLines*
  • SpatialMultiPoints*
  • SpatialPolygons*
  • SpatialGrid*
  • SpatialPixels*

Todas estas clases están acompañadas de un constructor que permite incluir los metadatos en forma de un data.frame. Por ejemplo, SpatialPointsDataFrame recibe como argumento adicional los metadatos para cada punto.

Coordinate Reference Systems

Es el modelo de representación de la tierra. El CRS incluye:

  • datum: Aproximación de la superficie terrestre
  • projection: Transformación de un sistema de coordenadas esférico a uno plano.
(example_crs <- CRS("+proj=longlat +datum=WGS84"))
## CRS arguments:
##  +proj=longlat +datum=WGS84 +ellps=WGS84 +towgs84=0,0,0

SpatialPoints

  • Por convención el formato default para coordenadas es (lon, lat)
  • Además de especificar las coordenadas necesitamos especificar un CRS
example_points <- data.frame(longitud = runif(10),   latitud = runif(10))
(spatial_points <- SpatialPoints(coords = example_points, proj4string = example_crs))
## class       : SpatialPoints 
## features    : 10

## coord. ref. : +proj=longlat +datum=WGS84 +ellps=WGS84 +towgs84=0,0,0

SpatialLines*

Para crear objetos de clase SpatialLines requerimos una lista de objetos clase Lines, que a su vez está compuesto por objetos de clase Line, que finalmente está compuesto por coordenadas que forman segmentos.

# Creamos 3 objetos Line y dos objetos Lines
line1 = Line(data.frame(runif(3), runif(3)))
line2 = Line(data.frame(runif(3), runif(3)))
line3 = Line(data.frame(c(0, 1), c(0, 1)))
lines1 <- Lines(list(line1, line2), ID = "A")
lines2 <- Lines(list(line3), ID = "B")

metadata <- data.frame(row.names = c("A", "B"), name = letters[1:2])
example_spatial_lines <- SpatialLines(list(lines1, lines2), example_crs)
(example_spatial_lines_dataframe <- SpatialLinesDataFrame(
  example_spatial_lines,
  metadata)
)
## class       : SpatialLinesDataFrame 
## features    : 2 
## extent      : 0, 1, 0, 1  (xmin, xmax, ymin, ymax)

## variables   : 1
## names       : name 
## min values  :    a 
## max values  :    b

SpatialPolygons*

Al igual que SpatialLines, SpatialPolygons se contruye con una lista de objetos clase Polygons, que a su vez se componen de objetos de clase Polygon, que son objetos de clase Line cuya coordenada final es igual a la coordenada inicial.

example_bounding_box <- data.frame(
  longitud = c(0, 1, 1, 0, 0),
  latitud = c(0, 0, 1, 1, 0)
)
example_polygon <- Polygon(example_bounding_box)
example_polygons <- Polygons(srl = list(example_polygon), "bounding box")
(example_spatial_polygons <- SpatialPolygons(list(example_polygons), proj4string = example_crs))
## class       : SpatialPolygons 
## features    : 1 
## extent      : 0, 1, 0, 1  (xmin, xmax, ymin, ymax)
## coord. ref. : +proj=longlat +datum=WGS84 +ellps=WGS84 +towgs84=0,0,0

GridTopology y SpatialGrid*

Para crear un objeto de clase SpatialGrid necesitamos un objeto clase GridTopology. El cual requiere básicamente, un centro y el tamaño de las celdas.

bb <- bbox(example_spatial_polygons)
cs <- c(0.1, 0.2)               # Tamaño de las celdas
cc <- bb[, 1] + cs / 2          # Cordenadas del centroide de inferior izquierdo de la grilla
cd <- ceiling(diff(t(bb))/cs)   # Total de celdas en cada dimensión
example_topology <- GridTopology(cellcentre.offset = cc, cellsize = cs, cells.dim = cd)
example_spatial_grid <- SpatialGrid(example_topology, example_crs)
summary(example_spatial_grid)
## Object of class SpatialGrid

##   min max

raster: Geographic Data Analysis and Modeling

Clases en raster

La librería da soporte a las clases:

  • RasterLayer
  • RasterBrick
  • RasterStack

Recomendamos esta librería para soporte de formatos raster, en lugar de SpatialGrid. La ventaja de usar la libraría raster es que los datos pueden ser analizados in-memory o leyendo los archivos del disco. Lo cual da ventaja en rendimiento para archivos grandes.

RasterLayer

Representa una sola capa de datos. Contiene información acerca del origen de los datos, el número de filas y columnas en la grilla.

(example_raster <- raster(
  nrows = 50,       # Número de filas
  ncols = 50,       # Número de columnas
  xmn = 0,        # Coordenada x mínima
  xmx = 1,         # Coordenada x máxima
  ymn = 0,         # Coordenada y mínima
  ymx = 1,          # Coordenada y máxima
  crs = example_crs, # Sistema de coordenadas
  vals = runif(50 ^ 2) * 255
))
## class       : RasterLayer 
## dimensions  : 50, 50, 2500  (nrow, ncol, ncell)
## resolution  : 0.02, 0.02  (x, y)
## extent      : 0, 1, 0, 1  (xmin, xmax, ymin, ymax)
## coord. ref. : +proj=longlat +datum=WGS84 +ellps=WGS84 +towgs84=0,0,0 
## data source : in memory
## names       : layer 
## values      : 0.005777293, 254.8167  (min, max)

RasterLayer (cont.)

RasterStack

Estas clases dan soporte cuando tenemos múltiples capas raster. La diferencia es que RasterBrick solo puede ser referenciado a un solo origen y los RasterStack a múltiples.

Los archivos RasterStack son listas de objetos RasterLayer que solo pueden mantenerse en RAM. En cambio, los objetos RasterBrick están contenidos en un solo archivo con múltiples bandas.

example_raster2 <- raster(
  nrows = 50, ncols = 50, xmn = 0, xmx = 1, ymn = 0, ymx = 1,
  crs = example_crs, vals = rgeom(50 ^ 2, 0.1)
)
example_raster3 <- raster(
  nrows = 50, ncols = 50, xmn = 0, xmx = 1, ymn = 0, ymx = 1,
  crs = example_crs, vals = rpois(50 ^ 2, 100)
)
(raster_stack <- stack(example_raster, example_raster2, example_raster3))

sf: Simple Features for R

Composición de simple features y clases en sf

Los simple feature son el estándar internacional de la representación y manipulación de datos espaciales. La ventaja de usar las simple features es intercompatibilidad entre lenguajes y bases de datos, además, el desempeño a mejorado bastante con respecto a sp (10x). Los tipos de datos más importantes soportados por sf son:

  • POINT
  • LINESTRING
  • POLYGON
  • MULTIPOINT
  • MULTILINESTRING
  • MULTIPOLYGON
  • GEOMETRYCOLLECTION

Clase sf

La clase sf extiende a un data.frame con una columna de single features. La columna de single features es un objeto de clase sfc (simple feature collection), que es una lista de objetos sfg (simple feature geometry).

example_point <- st_point(c(0, 1))
example_line <- st_linestring(matrix(c(0, 1, 0, 1), 2, 2))
outer = matrix(c(0, 0, 10, 0, 10, 10, 0, 10, 0, 0), ncol = 2, byrow = TRUE)
pts = list(outer)
example_polygon <- st_polygon(pts)
class(example_polygon)
## [1] "XY"      "POLYGON" "sfg"
(example_sfc <- st_sfc(example_polygon, example_line))
## Geometry set for 2 features 
## geometry type:  GEOMETRY
## dimension:      XY
## bbox:           xmin: 0 ymin: 0 xmax: 10 ymax: 10
## epsg (SRID):    NA
## proj4string:    NA

## LINESTRING(0 0, 1 1)
